# 14_传播模型适用性与公式修正

> 项目：RT-TETRA Cover Studio
>
> 版本：V1.0
>
> 状态：代码完成，待无线专业复核与现场校准

---

## 1. 任务目标

修正现有传播模型的适用范围、公式完整性和工程依据，使软件能够明确区分：

- 标准模型计算；
- 超出标准适用范围的计算；
- 依赖现场数据标定的工程模型。

本任务完成前，不将现有地面、隧道和高架覆盖距离作为最终工程定型结果。

---

## 2. 模型选型决定

### 2.1 800/860 MHz 地面场景

继续使用 COST231-Walfisch-Ikegami，并补齐 LOS/NLOS、街道方向修正、城市类型和基站相对屋顶高度的分段公式。

仅在模型允许的频率、距离、天线高度和城市环境范围内输出正式计算结果；超出范围时阻止计算或显示明确警告。

### 2.2 350/380/400 MHz 地面场景

不继续套用 COST231-WI，也不把 COST231-Hata 混入当前地面模型。

V1.0 采用独立的低频地面标定模型：

```text
PL(d) = PL(d0) + 10 × n × log10(d / d0) + K_cal
```

其中：

- `PL(d0)` 为参考距离实测路径损耗，或在缺少实测值时采用自由空间参考值；
- `n` 为路径损耗指数；
- `K_cal` 为线路或环境标定修正；
- 模型必须记录标定距离范围和数据来源；
- 未提供实测标定数据时，报告必须标记为“初步估算”。

选择该方案的原因是轨道交通 350/380/400 MHz 地面区段常见短距离、低基站天线条件，不应为了使用标准名称而套用超范围模型。Okumura-Hata 可作为后续候选，但其典型距离和基站高度范围与当前短距离、低天线算例并不完全匹配，V1.0 暂不启用。

### 2.3 地下场景

保留 ITU-R P.1238 形式。距离损耗系数、楼层损耗和墙体损耗必须说明取值来源；模型适用频率和距离范围应进入校验。

### 2.4 隧道场景

保留“自由空间损耗 + 单位距离衰减 + 附加损耗”的工程模型，但不再把当前截面修正系数视为已验证标准公式。

隧道模型的正式结果必须基于以下来源之一：

1. 现场测试数据回归；
2. 同类型线路经批准的设计参数；
3. 明确引用的技术标准或公开文献参考值。

未满足上述条件时，报告标记为“经验参数估算”。

### 2.5 高架场景

高架模型继续以地面基础模型叠加工程修正，但当前 `0.25 dB/m` 高度增益和 `500/R` 曲线损耗不得直接作为定型系数。

V1.0 将高架修正改为可追溯的标定修正值；高度和曲线半径保留为工程信息，不在没有数据依据时自动生成确定性增益或损耗。

## 3. COST231-WI 修正范围

实现必须覆盖：

- LOS 与 NLOS 两种公式；
- 自由空间损耗 `L0`；
- 屋顶到街道绕射与散射损耗 `Lrts`；
- 街道方向角修正 `Lori`；
- 多屏绕射损耗 `Lmsd`；
- `Lbsh`、`ka`、`kd`、`kf` 的完整分段条件；
- 大城市与中小城市/郊区系数选择；
- `Lrts + Lmsd <= 0` 时回退到 `L0`；
- 频率、距离、基站高度、移动台高度和建筑参数适用性校验；
- 超出适用范围时的错误或警告。

公式和常数必须逐项注明来源版本，不允许只写“工程经验公式”。

## 4. 计划新增参数

### 4.1 COST231-WI

| 参数 | 单位/类型 | 用途 |
| --- | --- | --- |
| `propagation_condition` | LOS/NLOS | 选择传播条件 |
| `street_orientation_deg` | degree | 计算 `Lori` |
| `city_type` | medium/metropolitan | 选择 `kf` |
| `model_correction_db` | dB | 实测校准修正 |

### 4.2 低频地面标定模型

| 参数 | 单位 | 用途 |
| --- | --- | --- |
| `reference_distance_m` | m | 参考距离 `d0` |
| `path_loss_exponent` | - | 距离损耗指数 `n` |
| `model_correction_db` | dB | 线路标定修正 `K_cal` |
| `calibration_min_distance_m` | m | 标定最小距离 |
| `calibration_max_distance_m` | m | 标定最大距离 |

### 4.3 隧道和高架

| 参数 | 单位 | 用途 |
| --- | --- | --- |
| `calibration_offset_db` | dB | 实测或设计标定修正 |
| `train_blockage_loss_db` | dB | 列车遮挡附加损耗 |
| `section_correction_db` | dB | 经依据确认的隧道截面修正 |
| `viaduct_correction_db` | dB | 经依据确认的高架综合修正 |

参数来源说明属于报告工程信息，不参与数值计算，但必须可追溯。

## 5. 参考点测试

新增测试不得只检查曲线单调性。每个模型至少包含：

| 测试类别 | 验证内容 |
| --- | --- |
| 标准公式参考点 | 使用人工独立计算的输入和期望 Path Loss |
| 分段边界 | 验证 LOS/NLOS、屋顶上下、城市类型等条件切换 |
| 适用范围边界 | 验证最低/最高频率、距离和天线高度 |
| 超范围输入 | 验证阻止计算或明确警告 |
| 自由空间下限 | 验证 COST231-WI 附加损耗不成立时回退 `L0` |
| 标定模型参考点 | 验证 `d=d0`、距离扩大 10 倍和修正值叠加 |
| 报告追溯 | 验证公式来源、适用范围和校准状态进入序列化及报告 |

参考值必须独立写入测试，不得先调用待测实现生成期望值。

## 6. 实施顺序

1. 建立失败测试，覆盖 COST231-WI 标准参考点和范围校验；
2. 完整实现 COST231-WI 公式；
3. 新增 350/380/400 MHz 低频地面标定模型；
4. 将地面场景改为按频率和模型选择显式切换，不静默套用；
5. 将隧道和高架经验系数改为有来源的标定参数；
6. 同步配置、示例、GUI、序列化、Word/PDF 报告和文档；
7. 执行完整回归和人工工程复核。

---

## 7. 验收标准

- 400 MHz 输入不再调用 COST231-WI；
- COST231-WI 标准参考点与独立计算值在规定容差内一致；
- 模型适用范围在计算前完成校验；
- 隧道和高架报告能够说明系数来源及是否已标定；
- 四类模型均有数值参考点、范围边界和异常输入测试；
- GUI、序列化、Word/PDF 报告展示相同的模型名称、公式和校准状态；
- 全量自动化测试通过后，才重新构建 Windows onedir 程序。
