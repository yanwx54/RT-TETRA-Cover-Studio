# 09_测试用例

> 项目：RT-TETRA Cover Studio
>
> 版本：V1.0 Draft
>
> 状态：初稿

---

## 1. 测试目标

V1.0 测试重点是保证计算正确、异常输入可控、核心流程可重复。

测试优先级：

1. 链路预算；
2. 参数校验；
3. 传播模型；
4. 覆盖距离求解；
5. 曲线数据；
6. 参数敏感性；
7. 报告导出；
8. GUI 交互。

---

## 2. 测试分类

| 类型 | 说明 |
| --- | --- |
| 单元测试 | 测试单个公式、模型、校验函数 |
| 集成测试 | 测试一次完整覆盖计算 |
| 边界测试 | 测试极限参数和异常输入 |
| 回归测试 | 确保已有算例结果不变 |
| UI 测试 | 测试输入、计算、展示和导出流程 |

---

## 3. 链路预算测试

| 编号 | 输入 | 预期 |
| --- | --- | --- |
| LB-001 | 基站 25 W、手台 1 W | 分别换算为 43.98 dBm、30.00 dBm |

## 11. 验收测试

V1.0 验收条件：

- 所有核心公式测试通过；
- 四类场景至少各有一个可运行算例；
- 一次完整计算时间不超过 3 秒；
- 结果对象包含计算过程；
- GUI 可完成输入、计算、展示、导出；
- 用户手册可指导用户完成一次计算。
