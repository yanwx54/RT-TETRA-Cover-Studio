# 06_项目目录规范

> 项目：RT-TETRA Cover Studio
>
> 版本：V1.0 Draft
>
> 状态：初稿

---

## 1. 目的

本文档定义项目目录用途、文件放置规则和后续开发约束，避免源码、文档、报告、配置混放。

---

## 2. 根目录结构

```text
RT-TETRA-Cover-Studio/
├── assets/
├── config/
├── docs/
├── examples/
├── packaging/
├── reports/
├── scripts/
├── src/
├── tests/
├── .gitignore
├── PROJECT_INDEX.md
├── README.md
└── requirements.txt
```

---

## 3. 根目录文件说明

| 文件 | 用途 |
| --- | --- |
| PROJECT_INDEX.md | 项目总控台，记录当前阶段、下一步和里程碑 |
| README.md | 项目说明，面向使用者和开发者 |
| requirements.txt | Python 依赖清单 |
| .gitignore | Git 忽略规则 |

根目录不放临时测试文件、报告输出、缓存文件。

## 5. src 目录

建议结构：

```text
src/
├── app/
├── common/
├── engine/
├── models/
├── report/
└── ui/
```

| 子目录 | 用途 |
| --- | --- |
| app | 业务编排 |
| common | 公共工具、常量、校验 |
| engine | 计算引擎 |
| models | 传播模型 |
| report | 报告导出 |
| ui | GUI |

---

## 6. tests 目录

建议结构：

```text
tests/
├── test_link_budget.py
├── test_models.py
├── test_engine.py
├── test_validation.py
└── test_report.py
```

测试文件必须能独立运行，不依赖 GUI 操作。

---

## 7. config 目录

存放配置文件：

- 默认参数；
- 频段列表；
- 覆盖等级阈值；
- 模型默认系数；
- 报告模板配置。

配置文件应使用明确格式，优先 JSON、YAML 或 TOML。

---

## 8. examples 目录

存放示例：

- 示例输入参数；
- 示例计算结果；
- 示例报告；
- 标准算例。

示例应可用于回归测试和用户学习。

---

## 9. reports 目录

存放本地生成报告。

要求：

- 报告输出不建议提交 Git；
- 文件名应包含日期、场景或项目名；
- 导出失败不能污染目录。

---

## 10. packaging 目录

存放打包配置文件：

- PyInstaller spec；
- 打包资源清单；
- 发布配置。

打包产物不得放入本目录，应输出到 `dist/` 并由 `.gitignore` 忽略。

---

## 11. assets 目录

存放：

- 图标；
- 图片；
- 报告模板资源；
- UI 静态资源。

不得存放运行日志和临时文件。

---

## 12. scripts 目录

存放辅助脚本：

- 环境检查；
- 文档生成；
- 测试运行；
- 打包发布。

脚本应在文件头说明用途和运行方式。

---

## 13. 命名规范

- 文档使用两位数字前缀；
- Python 文件使用小写加下划线；
- 测试文件以 `test_` 开头；
- 配置文件名体现用途；
- 生成文件避免覆盖源文件。

---

## 14. Git 忽略建议

建议忽略：

```text
__pycache__/
*.pyc
.venv/
venv/
.pytest_cache/
build/
dist/
reports/*
!reports/.gitkeep
*.log
```

正式规则应写入 `.gitignore`。
