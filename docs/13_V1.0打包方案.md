# 13_V1.0 打包方案

> 项目：RT-TETRA Cover Studio
>
> 版本：V1.0 Draft
>
> 状态：初稿

---

## 1. 打包目标

V1.0 打包目标是在 Windows 10/11 上生成可运行的桌面程序目录：

```text
dist/
└── RT-TETRA-Cover-Studio/
    └── RT-TETRA-Cover-Studio.exe
```

用户可双击 `RT-TETRA-Cover-Studio.exe` 启动 GUI，完成参数输入、计算、曲线查看和 Word/PDF 报告导出。

---

## 2. 打包方案

V1.0 使用 PyInstaller 的 `onedir` 模式。

选择原因：

- PySide6、Matplotlib、ReportLab 依赖资源较多，`onedir` 更稳定；
- 便于检查缺失 DLL、Qt 插件和字体资源；
- 便于 V1.0 验收阶段定位问题；
- 后续确认稳定后，再评估是否切换为 `onefile`。

---

## 3. 入口和资源

打包入口：

```text
scripts/run_gui.py
```

必须随程序携带的资源：

```text
config/
examples/
```

代码通过 `rt_tetra_cover_studio.paths.resource_path()` 定位资源；源码运行时读取项目根目录，打包运行时读取 PyInstaller 解包目录。

---

## 4. 构建命令

首次构建前安装依赖：

```powershell
python -m pip install -r requirements.txt
```

执行打包：

```powershell
.\scripts\build_windows.ps1
```

如果当前机器没有 `python` 命令，可显式指定解释器：

```powershell
.\scripts\build_windows.ps1 -Python "C:\Path\To\python.exe"
```

脚本会依次执行：

1. 安装依赖；
2. 编译检查；
3. 全量测试；
4. PyInstaller 打包；
5. 检查 exe 是否生成。

---

## 5. 验收检查

打包完成后检查：

- `dist/RT-TETRA-Cover-Studio/RT-TETRA-Cover-Studio.exe` 存在；
- 双击 exe 可打开 GUI；
- 可加载默认参数并完成一次计算；
- Path Loss / RSSI 曲线正常显示；
- Word 报告可导出并打开；
- PDF 报告可导出并打开；
- 报告中包含工程信息、页眉页脚和曲线图片。

---

## 6. Git 规则

提交 Git 的内容：

- `packaging/rt_tetra_cover_studio.spec`
- `scripts/build_windows.ps1`
- 打包方案文档

不提交 Git 的内容：

- `build/`
- `dist/`
- 本地生成报告
- 虚拟环境和缓存

---

## 7. 当前限制

V1.0 暂不处理：

- 安装包制作；
- 自动签名；
- 自动更新；
- 多平台打包；
- `onefile` 单文件发布。

这些内容进入后续版本或发布工程阶段。
