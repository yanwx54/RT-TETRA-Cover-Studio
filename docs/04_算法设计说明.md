## 1. 文档目的

本文档定义 V1.0 核心算法流程，包括链路预算、路径损耗、覆盖距离求解、RSSI 曲线、Path Loss 曲线和参数敏感性分析。

---

## 2. 总体算法流程

```text
输入参数
↓
参数标准化
↓
参数校验
↓
将基站和手台发射功率从 W 转换为 dBm
↓
计算双向 EIRP、最低要求接收功率和上下行 MAPL
↓
选择传播模型
↓
求解最大覆盖距离
↓
生成曲线数据
↓
执行敏感性分析
↓
输出 CalculationResult
```

## 3. 参数标准化

进入计算引擎前，所有单位必须统一。

| 参数 | 标准单位 |
| --- | --- |
| 频率 | MHz |
| 距离 | m |
| 用户输入功率 | W |
| 引擎内部功率 | dBm |
| 增益 | dBi |
| 损耗 | dB |
| 高度 | m |

标准化只做单位转换，不修改用户业务意图。

## 4. 参数校验算法

校验步骤：

1. 检查必填项；
2. 检查数值类型；
3. 检查工程合理范围；
4. 检查场景参数完整性；
5. 检查模型适用范围；
6. 返回错误列表。

有错误时不得进入计算。

## 5. 双向链路预算算法

功率换算：

```text
P_dBm = 10 × log10(P_W × 1000)
```

上下行 EIRP：

```text
EIRP_BS = P_BS_dBm + G_BS - L_feeder - L_other
EIRP_MS = P_MS_dBm + G_MS - L_body
```

边缘覆盖率按单侧正态分布地点概率计算：

```text
M_shadow = Z(p) × σ
```

其中 `p` 为边缘覆盖地点概率，V1.0 默认 95%，`Z(95%) = 1.645`。

最低要求接收功率：

```text
Req_Rx_MS = Sens_MS + L_body - G_MS + M_shadow + M_int + L_pen
Req_Rx_BS = Sens_BS - G_BS + L_feeder + L_other - G_div
            + M_shadow + M_int + L_pen
```

最大允许路径损耗：

```text
MAPL_DL = EIRP_BS - Req_Rx_MS
MAPL_UL = EIRP_MS - Req_Rx_BS
System_MAPL = min(MAPL_DL, MAPL_UL)
```

系统覆盖距离由较小 MAPL 对应的受限链路决定。基站与手台接收灵敏度分别输入并保留 dBm 单位。

## 5.1 地面模型选择与范围

- `ground_model=low_band`：仅允许 300–500 MHz，并在配置的标定距离范围内求解；
- `ground_model=cost231_wi`：仅允许 800–2000 MHz、20–5000 m、基站高度 4–50 m、移动台高度 1–3 m；
- 模型由用户显式选择，频率与模型不匹配时阻止计算；
- 求解器自动将搜索区间限制到模型适用范围；
- 未校准参数产生 warning，并在报告中输出参数来源。

算法输出必须保留：

- 公式；
- 参数代入；
- 计算结果；
- 单位。

## 6. 覆盖距离求解算法

目标：

```text
PathLoss(distance) = System_MAPL
```

V1.0 建议使用二分法。

步骤：

1. 设置 `low = 1 m`；
2. 设置 `high = 50000 m`；
3. 计算 `PathLoss(low)` 和 `PathLoss(high)`；
4. 若 `PathLoss(low) > MaxPathLoss`，返回无有效覆盖；
5. 若 `PathLoss(high) <= MaxPathLoss`，返回达到搜索上限；
6. 循环计算中点 `mid`；
7. 若 `PathLoss(mid) <= MaxPathLoss`，令 `low = mid`；
8. 否则令 `high = mid`；
9. 当 `high - low <= 1 m` 或达到最大迭代次数时停止；
10. 返回 `low` 作为最大覆盖距离。

默认参数：

| 参数 | 值 |
| --- | --- |
| 最小距离 | 1 m |
| 最大距离 | 50000 m |
| 距离精度 | 1 m |
| 最大迭代次数 | 100 |

## 7. RSSI 计算算法

RSSI：

```text
DL_RSSI = EIRP_BS - PathLoss(distance) + G_MS - L_body - L_pen
```

阴影衰落余量和干扰余量用于覆盖设计概率，不在瞬时 RSSI 中重复扣除。若系统为上行受限，下行 RSSI 曲线仅作为下行接收电平参考。

输出：

- 指定距离 RSSI；
- 覆盖边界 RSSI；
- 距离-RSSI 曲线数据。

覆盖等级建议：

| RSSI | 等级 |
| --- | --- |
| >= -85 dBm | 优 |
| -95 到 -85 dBm | 良 |
| -105 到 -95 dBm | 可用 |
| < -105 dBm | 边缘或不可用 |

阈值后续可配置化。

## 8. 曲线数据生成算法

曲线类型：

- 距离-Path Loss；
- 距离-RSSI。

步骤：

1. 以最大覆盖距离为上限；
2. 生成固定数量采样点；
3. 对每个距离调用传播模型；
4. 计算对应 RSSI；
5. 输出通用数据点列表。

V1.0 默认采样点数量为 50。

---

## 9. 参数敏感性分析算法

默认分析参数：

| 参数 | 调整方式 |
| --- | --- |
| 发射功率 | +3 dB |
| 天线增益 | +2 dBi |
| 基站高度 | +5 m |
| 阴影衰落标准差 | +1 dB |
| 边缘覆盖率 | 提高 1 个百分点 |

步骤：

1. 计算基准覆盖距离；
2. 复制输入参数；
3. 修改一个参数；
4. 重新计算覆盖距离；
5. 记录变化量；
6. 还原参数；
7. 输出敏感性列表。

敏感性分析必须复用计算引擎，不允许另写简化公式代替。

---

## 10. 边界处理

| 情况 | 处理方式 |
| --- | --- |
| 最小距离已不满足覆盖 | 返回无有效覆盖 |
| 最大距离仍满足覆盖 | 返回达到搜索上限 |
| 模型返回非法值 | 中止并返回计算异常 |
| 迭代未收敛 | 返回 warning 和最后结果 |
| 参数缺失 | 阻止计算 |

---

## 11. 精度要求

- 距离结果保留到 1 m；
- 损耗结果保留到 0.01 dB；
- RSSI 结果保留到 0.01 dBm；
- 测试断言使用容差，不直接比较浮点完全相等。
