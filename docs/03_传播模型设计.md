# 03_传播模型设计

> 项目：RT-TETRA Cover Studio
>
> 版本：V1.0 Draft
>
> 状态：初稿

---

## 1. 文档目的

本文档定义 V1.0 使用的传播模型、适用场景、输入参数、输出结果和实现约束。

传播模型模块只负责计算指定距离下的路径损耗，不负责 GUI 展示、报告导出和覆盖距离迭代。

---

## 2. V1.0 模型范围

| 场景 | 模型 | 状态 |
| --- | --- | --- |
| 地下站厅、站台 | ITU Indoor | V1.0 支持 |
| 隧道 | Tunnel Model | V1.0 支持 |
| 350/380/400 MHz 地面 | Low-Band Calibrated Ground | V1.0 支持，默认参数未校准 |
| 800/860 MHz 城市地面 | COST231-Walfisch-Ikegami | V1.0 支持 |
| 高架 | 所选地面模型 + Viaduct Calibration | V1.0 支持，需校准来源 |

地面和高架场景必须显式选择低频标定模型或 COST231-WI，不根据频率静默套用模型。

---

## 3. 统一模型接口

所有传播模型必须实现统一接口：

```python
def calculate_path_loss(input_data, distance_m):
    """返回指定距离下的路径损耗结果。"""
```

统一输入：

| 参数 | 说明 |
| --- | --- |
| frequency_mhz | 工作频率，MHz |
| distance_m | 计算距离，m |
| base_height_m | 基站天线高度，m |
| mobile_height_m | 手台高度，m |
| scenario_params | 场景专用参数 |

统一输出：

| 字段 | 说明 |
| --- | --- |
| model_name | 模型名称 |
| path_loss_db | 路径损耗，dB |
| formula_source | 公式来源 |
| intermediate_values | 中间计算量 |
| warnings | 模型适用性警告 |

---

## 4. 自由空间损耗

自由空间损耗作为多个模型的基础项。

```text
FSPL = 32.44 + 20log10(f_MHz) + 20log10(d_km)
```

参数：

- `f_MHz`：频率，MHz；
- `d_km`：距离，km。

要求：

- 距离必须大于 0；
- 频率必须大于 0；
- 输出单位为 dB。

---

## 5. ITU Indoor 模型

适用场景：

- 地下站厅；
- 地下站台；
- 室内公共区。

建议公式：

```text
L = 20log10(f_MHz) + Nlog10(d_m) + Lf(n) - 28
```

参数：

| 参数 | 说明 |
| --- | --- |
| f_MHz | 频率，MHz |
| d_m | 距离，m |
| N | 距离功率损耗系数 |
| Lf(n) | 楼层或隔墙附加损耗 |

V1.0 默认约束：

- 地下站厅、站台可先不考虑楼层穿透；
- 墙体材料影响进入 `scenario_params`；
- 阴影衰落、干扰和穿透等链路预算项不在传播模型内处理。

---

## 6. Tunnel Model

适用场景：

- 单洞隧道；
- 区间隧道；
- 近似规则断面的轨道交通隧道。

V1.0 使用可追溯工程模型：

```text
L = FSPL + alpha*d_km + K_section + L_bend + L_train + K_cal
```

参数：

| 参数 | 说明 |
| --- | --- |
| FSPL | 自由空间损耗 |
| alpha | 隧道单位距离附加损耗，dB/km |
| d_km | 距离，km |
| K_section | 经依据确认的隧道断面修正项 |
| L_bend | 弯曲附加损耗 |
| L_train | 列车遮挡附加损耗 |
| K_cal | 现场或批准设计参数校准值 |

场景参数：

- 隧道宽度；
- 隧道高度；
- 隧道类型；
- 是否弯曲。

`calibration_status=unverified` 时结果只作为初步估算，报告必须输出 `calibration_source`。

---

## 7. COST231-Walfisch-Ikegami 模型

适用场景和范围：

- 地面区段；
- 城市轨道交通沿线；
- 建筑物影响明显的地面环境。
- 频率 800–2000 MHz；
- 距离 20–5000 m；
- 基站高度 4–50 m；
- 移动台高度 1–3 m。

模型结构：

```text
LOS: L = 42.6 + 26log10(d_km) + 20log10(f_MHz)
NLOS: L = L0 + max(Lrts + Lmsd, 0)
```

参数：

| 参数 | 说明 |
| --- | --- |
| L0 | 自由空间损耗 |
| Lrts | 屋顶到街道绕射及散射损耗 |
| Lmsd | 多重屏障绕射损耗 |

场景参数：

- 建筑平均高度；
- 建筑间距；
- 街道宽度；
- 基站高度；
- 手台高度。
- LOS/NLOS；
- 街道方向角；
- 城市类型；
- 模型校准修正。

实现要求：

- 将各损耗项拆分保存到 `intermediate_values`；
- 实现 `Lori` 和 `Lbsh/ka/kd/kf` 全部分段条件；
- 参数超出模型适用范围时阻止计算；
- 附加损耗小于 0 时回退自由空间损耗 `L0`；
- 不在模型内处理阴影衰落余量、接收灵敏度或上下行 MAPL。

### 7.1 低频地面标定模型

350/380/400 MHz 地面场景采用：

```text
L = FSPL(f,d0) + 10nlog10(d/d0) + K_cal
```

模型参数包括参考距离、路径损耗指数、校准修正、标定距离范围、校准状态和参数来源。默认参数未经过现场数据校准，因此结果标记为初步估算。

## 8. 高架修正模型

高架场景基于用户明确选择的地面模型增加标定修正：

```text
L_viaduct = L_ground + K_viaduct + L_train + K_cal
```

修正项考虑：

- 高架高度；
- 曲线半径；
- 遮挡程度；
- 天线相对位置。

高架高度和曲线半径作为工程信息保留，不再自动生成 `0.25 dB/m` 或 `500/R` 修正。`K_viaduct` 必须由现场数据、批准设计参数或明确文献给出。

---

## 9. 模型选择规则

| scenario_type | 模型 |
| --- | --- |
| underground | ITU Indoor |
| tunnel | Tunnel Model |
| ground | 显式选择 Low-Band Calibrated Ground 或 COST231-WI |
| viaduct | 显式选择地面基础模型 + Viaduct Calibration |

模型选择由计算引擎完成，不允许 GUI 直接实例化传播模型。

---

## 10. 测试要求

每个模型至少包含：

- 正常参数测试；
- 距离为 0 或负数测试；
- 频率非法测试；
- 场景参数缺失测试；
- 中间计算项完整性测试；
- 单调性测试：距离增加时 Path Loss 不应下降。
- 独立数值参考点测试；
- LOS/NLOS、街道方向角和屋顶高度分段测试；
- 适用范围边界和超范围测试；
- 校准状态与参数来源序列化/报告测试。
