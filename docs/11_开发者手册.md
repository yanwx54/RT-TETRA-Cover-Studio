## 3. V1.0 技术栈

V1.0 技术栈已确定。

| 项目 | 选型 | 用途 |
| --- | --- | --- |
| Python 版本 | Python 3.13 | 核心语言和运行环境 |
| GUI 框架 | PySide6 + Qt Widgets | Windows 桌面界面 |
| 绘图库 | Matplotlib | RSSI 和 Path Loss 曲线 |
| Word 导出 | python-docx | 生成 Word 报告 |
| PDF 导出 | ReportLab | 生成 PDF 报告 |
| 测试框架 | pytest | 单元测试和集成测试 |
| 打包工具 | PyInstaller | 生成 Windows 可运行程序目录 |

开发环境初始化：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

当前机器若没有 `python` 命令，应先安装 Python 3.13，并确认 PowerShell 可识别 `python --version`。

## 4. 推荐实现顺序

1. 定义数据结构；
2. 实现参数校验；
3. 实现链路预算；
4. 实现一个传播模型；
5. 实现覆盖距离求解；
6. 生成计算结果对象；
7. 增加测试；
8. 扩展全部传播模型；
9. 实现曲线数据；
10. 实现 GUI；
11. 实现报告导出；
12. 建立打包方案；
13. 执行 V1.0 验收。

## 5. 模块边界

| 模块 | 可以做 | 不可以做 |
| --- | --- | --- |
| ui | 收集输入、展示结果 | 写计算公式 |
| app | 编排流程 | 实现传播模型 |
| engine | 调度计算 | 操作 GUI |
| models | 计算路径损耗 | 导出报告 |
| report | 生成报告 | 重新计算 |
| common | 校验、单位、常量 | 依赖业务模块 |

## 6. 数据结构要求

核心对象：

- `CalculationInput`
- `LinkBudgetResult`
- `PropagationResult`
- `CurvePoint`
- `CalculationResult`

要求：

- 字段名带单位；
- 结果保留中间过程；
- 报告和 GUI 使用同一结果对象；
- 不在多个模块中重复定义同一数据结构。

---

## 7. 测试要求

新增核心逻辑必须有测试。

建议测试命名：

```text
tests/test_link_budget.py
tests/test_validation.py
tests/test_models.py
tests/test_engine.py
tests/test_report.py
```

测试重点：

- 正常参数；
- 边界参数；
- 非法参数；
- 浮点容差；
- 异常分支；
- 一次完整计算。

---

## 8. 计算实现要求

- 所有功率、增益、损耗使用 dB 体系；
- 距离进入模型前统一单位；
- 工程裕度只在链路预算中扣除；
- 传播模型只返回路径损耗；
- 覆盖距离求解必须记录迭代过程；
- 曲线数据从同一模型计算得到。

---

## 9. 报告实现要求

报告模块输入只能是 `CalculationResult`。

报告不得：

- 重新计算 EIRP；
- 重新调用传播模型；
- 修改计算结果；
- 静默覆盖文件。

---

## 10. 异常处理要求

异常信息应清晰：

- 字段名；
- 错误原因；
- 允许范围；
- 修复建议。

内部日志可记录技术细节，普通用户界面不显示堆栈。

## 12. Windows 打包

V1.0 使用 PyInstaller `onedir` 模式生成 Windows 可运行程序。

打包配置：

```text
packaging/rt_tetra_cover_studio.spec
```

构建命令：

```powershell
.\scripts\build_windows.ps1
```

构建脚本会先安装依赖、运行编译检查和全量测试，再执行 PyInstaller。打包产物输出到：

```text
dist/RT-TETRA-Cover-Studio/RT-TETRA-Cover-Studio.exe
```

源码运行和打包运行都应通过 `rt_tetra_cover_studio.paths.resource_path()` 读取 `config/`、`examples/` 等资源，不应在业务代码中直接依赖源码目录层级。
